Opciones de migración a HTML plano modularizado:

1. Includes vía JavaScript (fetch/insert):
   Pros:
   - Permite reutilizar fragmentos HTML (header, footer, sidebar, etc.)
   - No requiere frameworks ni herramientas externas
   - Compatible con cualquier servidor estático
   Contras:
   - No hay recarga parcial real (como en React/Vue)
   - SEO puede verse afectado si el contenido se inserta dinámicamente
   - Depuración más compleja si hay errores de carga

2. Copia/pega de componentes en cada página:
   Pros:
   - 100% compatible con HTML puro
   - No requiere JS adicional
   Contras:
   - Difícil de mantener si hay muchos cambios en los componentes
   - Propenso a errores por duplicación

3. Uso de plantillas server-side (no implementado aquí):
   Pros:
   - Permite máxima reutilización y control
   Contras:
   - Requiere backend (Node, PHP, etc.)

En este proyecto se utiliza la opción 1: includes vía JavaScript para mantener modularidad y compatibilidad.
